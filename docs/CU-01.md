# CU-01 - Registrarse en el sistema 
- Actor: Usuario 
- Precondiciones: El usuario no debe tener una cuenta registrada previamente. 
### Flujo principal:
1.  El usuario accede a la página de registro.
2. Ingresa nombre, apellido, email, nombre de usuario y contraseña.
3. El sistema valida que el email y el nombre de usuario no estén en uso.
4. El sistema crea la cuenta y redirige al usuario a su página principal. 
### Flujos alternativos:
- Si el email o usuario ya existe, el sistema muestra un mensaje de error indicando el campo duplicado.
- Si algún campo obligatorio está vacío, el sistema solicita completarlo antes de continuar. Postcondición: El usuario queda registrado y puede iniciar sesión.


## Flujo principal
Precondición: El usuario no debe tener una cuenta registrada previamente.

### Pasos
1. El usuario accede a la página de registro.
2. El usuario ingresa nombre, apellido, email, nombre de usuario y contraseña.
3. El sistema valida que el email y el nombre de usuario no estén en uso.
4. El sistema valida que todos los campos obligatorios estén completos.
5. El sistema crea la cuenta y redirige al usuario a su página principal
6. FIN CASO DE USO.

Postcondición: El usuario queda registrado en el sistema y puede iniciar sesión.


## Curso Alternativo
### Acciones
3a. Si el email ya está registrado, el sistema muestra un mensaje de error indicando el campo duplicado.
3b. Si el nombre de usuario ya está en uso, el sistema muestra un mensaje indicando que debe elegir otro.

Postcondición: El usuario queda registrado en el sistema y puede iniciar sesión.
