# CU-01 - Registrarse en el sistema 
- Actor: Usuario 
- Precondiciones: El usuario no debe tener una cuenta registrada previamente. 
### Flujo principal:
1.  El usuario accede a la página de registro.
2. Ingresa nombre, apellido, email, nombre de usuario y contraseña.
3. El sistema valida que el email y el nombre de usuario no estén en uso.
4. El sistema crea la cuenta y redirige al usuario a su página principal. 
### Flujos alternativos:
- Si el email o usuario ya existe, el sistema muestra un mensaje de error indicando el campo duplicado.
- Si algún campo obligatorio está vacío, el sistema solicita completarlo antes de continuar. Postcondición: El usuario queda registrado y puede iniciar sesión.


## Flujo principal
Precondición: El usuario no debe tener una cuenta registrada previamente.

### Pasos
1. El usuario accede a la página de registro.
2. El usuario ingresa nombre, apellido, email, nombre de usuario y contraseña.
3. El sistema valida que el email y el nombre de usuario no estén en uso.
4. El sistema valida que todos los campos obligatorios estén completos.
5. El sistema crea la cuenta y redirige al usuario a su página principal
6. FIN CASO DE USO.

Postcondición: El usuario queda registrado en el sistema y puede iniciar sesión.


## Curso Alternativo
### Acciones
3a. Si el email ya está registrado, el sistema muestra un mensaje de error indicando el campo duplicado.
3b. Si el nombre de usuario ya está en uso, el sistema muestra un mensaje indicando que debe elegir otro.

Postcondición: El usuario queda registrado en el sistema y puede iniciar sesión.



# Clase de analisis

## IURegistro

### Atributos
- datosFormulario
- errorVisible : Boolean

### Responsabilidades
- Mostrar el formulario de registro.
- Capturar nombre, apellido, email, usuario y contraseña.
- Validar campos obligatorios completos.
- Mostrar errores de duplicado o campo vacío.
- Redirigir al usuario tras el registro exitoso.

### Colaboradores
- RegistroController
- Usuario (actor)

### Requisitos Especiales
- RE-F01: Validar campos obligatorios antes de enviar.
- RE-F02: Mostrar el campo específico que tiene error de duplicado.


---

## RegistroController

### Atributos
- datosRegistro
- resultadoValidacion

### Responsabilidades
- Recibir los datos del formulario.
- Validar unicidad de email y nombre de usuario.
- Crear la cuenta y persistirla.
- Retornar resultado al formulario.

### Colaboradores
- IURegistro
- Usuario (Entity)

### Requisitos Especiales
- RE-C01: Verificar unicidad consultando la base antes de crear.
- RE-C02: Retornar mensaje de error específico del campo duplicado.


---

## Usuario

### Atributos
- idUsuario : String
- nombre : String
- apellido : String
- email : String
- nombreUsuario : String
- contraseña : String

### Responsabilidades
- Almacenar los datos de la cuenta.
- Garantizar unicidad de email y usuario.
- Persistir el alta en la base de datos.
- Confirmar la creación al controlador.

### Colaboradores
- RegistroController

### Requisitos Especiales
- RE-E01: El campo email debe tener restricción de unicidad a nivel de base de datos.
- RE-E02: La contraseña debe persistirse con hash, no en texto plano.


# Diagrama de Clases de Diseño

## Clase: FormularioRegistro «view»

### Atributos
- nombre : String
- apellido : String
- email : String
- nombreUsuario : String
- contraseña : String
- mensajeError : String

### Métodos
- mostrarFormulario() : void
- capturarDatos() : void
- mostrarError(msg : String) : void
- mostrarExito() : void
- redirigirInicio() : void

### Rol
Representa la interfaz de usuario encargada de mostrar el formulario de registro, capturar los datos ingresados y comunicar el resultado de la operación al usuario.


---

## Clase: RegistroController «controller»

### Atributos
- datosFormulario : Map
- resultadoValidacion : Boolean

### Métodos
- registrarUsuario(datos : Map) : void
- validarDatos(datos : Map) : Boolean
- verificarEmailUnico(email : String) : Boolean
- verificarUsuarioUnico(usr : String) : Boolean
- crearCuenta(datos : Map) : void
- retornarResultado(ok : Boolean) : void

### Rol
Coordina el proceso de registro. Recibe los datos desde la vista, valida la información, verifica la unicidad del email y del nombre de usuario, crea la cuenta y devuelve el resultado a la interfaz.

## Relaciones entre clases

### FormularioRegistro → RegistroController

#### Tipo
Dependencia («usa»)

#### Descripción
La vista utiliza al controlador para iniciar el proceso de registro.

```text
FormularioRegistro
        |
        | usa
        v
RegistroController
```
### RegistroController → Usuario
#### Tipo
Asociación («actualiza»)
#### Descripción
El controlador crea y actualiza instancias de Usuario durante el proceso de registro.
```text
RegistroController
        |
        | actualiza
        v
Usuario
```
### Flujo General
```text
FormularioRegistro (View)
          |
          v
RegistroController (Controller)
          |
          v
Usuario (Model)
```

1. El usuario completa el formulario.
2. FormularioRegistro captura los datos.
3. RegistroController recibe y valida la información.
4. RegistroController verifica unicidad de email y usuario.
5. RegistroController crea o actualiza un Usuario.
6. RegistroController devuelve el resultado a la vista.
7. FormularioRegistro muestra éxito o error al usuario.
