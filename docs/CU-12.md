# CU-12 - Comentar fotos
- Actor: Usuario 
- Precondiciones:  El usuario debe tener acceso a la foto (propia o de un amigo con permiso habilitado)
### Flujo principal:
1. El usuario accede a una foto.
2. Escribe un comentario y lo envía.
3. El sistema publica el comentario asociado a la foto.
4. El usuario puede editar o eliminar sus propios comentarios en cualquier momento.

### Flujos alternativos:
- Si el usuario no tiene permiso para comentar en esa foto, la opción no aparece disponible.
- Si el comentario está vacío, el sistema no permite publicarlo. 


Postcondición: El comentario queda publicado y visible para quienes tengan acceso a esa foto.

## Flujo principal
Precondición: El usuario debe estar registrado e iniciada la sesión. 

### Pasos
1. El usuario accede a una foto dentro de un álbum.
2. El sistema verifica que el usuario tiene permiso para comentar.
3. El usuario escribe un comentario en el campo habilitado y lo envía.
4. El sistema publica el comentario asociado a la foto.
5. El comentario queda visible para los usuarios con acceso al álbum.
6. FIN CASO DE USO.



## Curso Alternativo
### Acciones
2a. Si el usuario no tiene permiso para comentar en ese álbum, la opción de comentar no aparece disponible.
3a. Si el comentario está vacío, el sistema no permite publicarlo y muestra un aviso.
4a. El usuario puede editar sus propios comentarios: el sistema actualiza el comentario y registra la modificación.
4b. El usuario puede eliminar sus propios comentarios: el sistema elimina el comentario del álbum.
4c. El dueño del álbum puede eliminar comentarios de sus amigos en sus fotos.

Postcondición: El comentario queda publicado y visible para quienes tengan acceso al álbum que contiene la foto.


# Clase de analisis

## IUFoto

### Atributos
- idFotoMostrada
- textoComentario (input)
- botonEnviar (estado: visible/oculto según permisos)
- botonEditar / botonEliminar (por comentario propio)

### Responsabilidades
- Mostrar la foto y sus comentarios.
- Mostrar/ocultar el campo de comentario según permisos.
- Capturar el texto y disparar el envío.
- Mostrar avisos al usuario (vacío, error).
- Ofrecer editar/eliminar solo en comentarios propios.

### Colaboradores
- ComentarController

### Requisitos Especiales
- RE-02: Los avisos al usuario (comentario vacío, error de publicación) no deben bloquear la navegación dentro del álbum.
- RE-03: Las acciones editar/eliminar deben estar disponibles solo en los comentarios cuyo autor sea el usuario actual.
- RE-04: La opción de eliminar comentarios ajenos debe estar visible únicamente cuando el usuario actual sea dueño del álbum que contiene la foto.


---

## ComentarController

### Atributos
- foto: referencia a Foto
- usuarioActual: referencia a Usuario
- comentarioEnEdicion: referencia a Comentario

### Responsabilidades
- Validar permisos del usuario sobre la foto.
- Validar que el comentario no esté vacío.
- Crear el Comentario y asociarlo a la Foto.
- Actualizar o eliminar comentarios.
- Permitir al dueño del álbum eliminar comentarios ajenos.

### Colaboradores
- IUFoto
- Foto
- Comentario

### Requisitos Especiales
- RE-01: La validación de permisos debe consultarse a Foto antes de cualquier escritura de Comentario.
- RE-02: La edición debe actualizar fechaModificación del Comentario en la misma transacción.
- RE-03: El borrado de un comentario ajeno solo es válido si el usuario actual es dueño del álbum que contiene la foto.
- RE-04: Las operaciones de creación, edición y eliminación deben quedar registradas para auditoría (autor, fecha, acción).


---

## Foto

### Atributos
- idFoto
- album (referencia)
- dueño
- permisosDeComentario (lista de usuarios autorizados)

### Responsabilidades
- Conocer su dueño y su álbum.
- Indicar quién puede comentarla (según permisos del álbum).
- Mantener la lista de Comentarios asociados.

### Colaboradores
- ComentarController

### Requisitos Especiales
- Los permisos de comentario se heredan del álbum que la contiene.
- No exponer la lista de comentarios a quien no tenga acceso al álbum.


---

## Comentario

### Atributos
- idComentario
- texto
- autor
- foto (referencia)
- fechaCreación
- fechaModificación

### Responsabilidades
- Conservar su autor, texto y fechas.
- Permitir actualizar su texto (registrando fechaModificación).
- Permitir eliminarse del álbum.

### Colaboradores
- ComentarController
- Foto

### Requisitos Especiales
- El borrado es físico y definitivo (no se conservan eliminados).
- fechaCreación es inmutable.
- fechaModificación se actualiza ante cualquier cambio en el texto.


---

# Requisitos Especiales Generales del CU-12

- RE-01: El sistema debe responder a la publicación de un comentario en menos de 2 segundos.
- RE-02: Todo comentario debe quedar registrado con autor, fecha de creación y fecha de última modificación.
- RE-03: La verificación de permisos debe ser previa al renderizado del campo de comentario.
- RE-04: Los comentarios eliminados no se conservan; el borrado es físico y definitivo.

## Relaciones entre clases

### DetallesFoto → ComentarioController

#### Tipo
Dependencia («usa»)

#### Descripción
La vista utiliza al controlador para publicar, editar o eliminar comentarios.

```text
DetallesFoto
      |
      | usa
      v
ComentarioController
```
### ComentarioController → Usuario
#### Tipo
Asociación («verifica»)
#### Descripción
El controlador consulta al usuario para verificar permisos antes de permitir acciones sobre comentarios.
```text
ComentarioController
        |
        | verifica
        v
Usuario
```
### ComentarioController → Foto
#### Tipo
Asociación («obtiene»)
#### Descripción    
El controlador consulta la foto para validar permisos y recuperar información relacionada con los comentarios.
```text
ComentarioController
        |
        | obtiene
        v
Foto
```
### ComentarioController → Comentario
#### Tipo
Asociación («gestiona»)
#### Descripción
El controlador crea, actualiza y elimina instancias de Comentario.
```text
ComentarioController
        |
        | gestiona
        v
Comentario
```
### Flujo General
```text
DetallesFoto (View)
          |
          v
ComentarioController (Controller)
        /     |      \
       /      |       \
      v       v        v
 Usuario    Foto   Comentario
 (Model)   (Model)   (Model)
```
1. El usuario visualiza una foto y sus comentarios.
2. DetallesFoto captura el comentario ingresado.
3. ComentarioController valida el texto.
4. ComentarioController verifica permisos del usuario.
5. ComentarioController consulta la foto asociada.
6. Si la operación es válida, crea, edita o elimina un Comentario.
7. El controlador devuelve el resultado a la vista.
8. DetallesFoto actualiza la visualización y notifica al usuario.

### Patrón aplicado
Este diagrama implementa el patrón MVC (Model - View - Controller):
- View: DetallesFoto
- Controller: ComentarioController
- Model: Usuario, Foto, Comentario
La vista delega toda la lógica de negocio al controlador, mientras que el controlador interactúa con los modelos para validar permisos y gestionar comentarios.
