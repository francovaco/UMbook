# CU-03 - Buscar usuarios 
- Actor: Usuario 
- Precondiciones: El usuario debe estar registrado e iniciada la sesión. 
### Flujo principal:
1. El usuario accede al buscador de usuarios.
2. Ingresa un nombre o apellido.
3. El sistema muestra una lista de usuarios que coinciden con la búsqueda.
4. Cada resultado incluye un link para enviar una solicitud de amistad. 
### Flujos alternativos:
- Si no se encuentran resultados, el sistema informa que no hay usuarios con ese nombre o apellido.

Postcondición: El usuario puede visualizar los resultados y decidir si enviar una solicitud.

## Flujo principal
Precondición: El usuario debe estar registrado e iniciada la sesión. 

### Pasos
1. El usuario accede al buscador de usuarios.
2. El usuario ingresa un nombre o apellido en el campo de búsqueda.
3. El sistema busca entre todos los usuarios registrados.
4. El sistema muestra una lista de usuarios que coinciden con la búsqueda.
5. Cada resultado incluye un link para enviar una solicitud de amistad (extiende CU: Invitar Amigo).
6. FIN CASO DE USO.

## Curso Alternativo
### Acciones
2a. Si el campo de búsqueda está vacío, el sistema no realiza la búsqueda y solicita ingresar un término.
4a. Si no se encuentran resultados, el sistema informa que no hay usuarios con ese nombre o apellido.

Postcondición: El usuario visualiza los resultados de la búsqueda y puede decidir enviar una solicitud de amistad


# Clase de analisis

## IUBuscarUsuarios

### Atributos
- criteriosBusqueda
- resultadosVisible : Boolean

### Responsabilidades
- Mostrar el campo de búsqueda.
- Capturar nombre o apellido ingresado.
- Deshabilitar búsqueda si el campo está vacío.
- Mostrar lista de resultados coincidentes.
- Mostrar mensaje claro si no hay resultados.
- Ofrecer link para enviar solicitud de amistad.

### Colaboradores
- BuscarUsuariosController
- Usuario (Actor)

### Requisitos Especiales
- RE-F01: Deshabilitar la búsqueda si el campo está vacío.
- RE-F02: Mostrar mensaje claro cuando no hay resultados.


---

## BuscarUsuariosController

### Atributos
- criterioBusqueda
- resultadosBusqueda

### Responsabilidades
- Recibir el criterio de búsqueda.
- Consultar usuarios por nombre o apellido.
- Retornar lista de coincidencias a la interfaz.
- Retornar respuesta vacía si no hay resultados.

### Colaboradores
- IUBuscarUsuarios
- Usuario (Entity)

### Requisitos Especiales
- RE-C01: Buscar coincidencias parciales en nombre y apellido.
- RE-C02: No exponer datos privados en los resultados.


---

# Diagrama de Clases de Diseño

Representa la interfaz de usuario encargada de capturar el criterio de búsqueda, mostrar los resultados encontrados y comunicar mensajes o errores al usuario.


---

## Clase: BusquedaController «controller»

### Atributos

Coordina el proceso de búsqueda de usuarios. Recibe el término ingresado por la vista, valida el criterio, consulta los usuarios coincidentes y devuelve los resultados a la interfaz.

## Relaciones entre clases

### BuscadorUsuarios → BusquedaController

#### Tipo
Dependencia («usa»)

#### Descripción
La vista utiliza al controlador para ejecutar una búsqueda a partir del término ingresado por el usuario.

```text
BuscadorUsuarios
        |
        | usa
        v
BusquedaController
```

### BusquedaController → Usuario
#### Tipo
Asociación («consulta»)
#### Descripción
El controlador consulta la entidad Usuario para obtener las coincidencias que satisfacen el criterio de búsqueda.
```text
BusquedaController
        |
        | consulta
        v
Usuario
```
### Flujo General
```text
BuscadorUsuarios (View)
            |
            v
BusquedaController (Controller)
            |
            v
Usuario (Model)
```
1. El usuario ingresa un nombre o apellido en el buscador.
2. BuscadorUsuarios captura el término ingresado.
3. BusquedaController valida que el criterio sea válido.
4. BusquedaController consulta a Usuario mediante la operación de búsqueda.
5. Usuario retorna la lista de coincidencias encontradas.
6. BusquedaController devuelve los resultados a la vista.
7. BuscadorUsuarios muestra la lista de usuarios encontrados o informa que no existen resultados.



### Patrón aplicado

Este diagrama implementa el patrón **MVC (Model - View - Controller)**:

- **View:** BuscadorUsuarios
- **Controller:** BusquedaController
- **Model:** Usuario

La vista nunca consulta directamente al modelo; toda interacción pasa por el controlador.
